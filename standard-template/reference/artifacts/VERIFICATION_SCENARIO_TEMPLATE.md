## Quick Use Rule

- Planner는 packet 작성 시 최소한 위 5개 칸을 검토한다.
- Developer는 self-check에 이 틀을 사용한다.
- Tester는 검증 결과를 이 틀에 맞춰 기록한다.
- Reviewer는 빈칸이나 약한 시나리오를 finding으로 본다.
