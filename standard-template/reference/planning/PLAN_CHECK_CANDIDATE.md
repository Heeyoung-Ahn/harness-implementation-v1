# Plan Check Candidate

> HUMAN STATUS SUMMARY. This is a non-gating checklist candidate for planning quality. It does not block workflow or replace validator evidence.

## Purpose
- Provide a lightweight pre-implementation planning checklist without introducing a new mandatory gate in core.

## Candidate Checks
- Project name is explicitly recorded.
- In-scope and out-of-scope sections both exist.
- Open questions have an owner.
- Approval boundary is explicit.
- Active profile candidates and actual active profile state do not conflict.
- Downstream hold conditions are explicit when implementation must wait.

## Usage Rule
- Use this as an optional planner/reference checklist.
- Do not treat missing items here as an automatic validation failure in `OPS-29`.
- If repeated downstream evidence shows this should become a command or warning-only CLI check, open a separate follow-up packet.
