# Candidate Promotion Playbook

## Purpose

Move a harness item from candidate status to accepted standard output without polluting the accepted corpus.

## Procedure

1. Identify the candidate item and the exact reason it may be useful.
2. Check whether it is necessary to the harness philosophy.
3. Reject it immediately if it is redundant or outside the operating model.
4. If it is useful but incomplete, correct and adapt it for this project.
5. Resolve conflicts against current accepted outputs.
6. Run the promotion gate checklist.
7. Promote it into the correct category under `standard-harness\` only after the checklist passes.

## Do Not Skip

- project-specific adaptation
- wording cleanup
- structure normalization
- conflict resolution
- verification against the current accepted harness

## Rule

Never promote an item just because it is probably useful later.

Promote it only when it is already ready.
